IT FAQ - Virtualizacija in Omrežja

VIRTUALIZACIJA ▼
Kaj je virtualizacija?

Virtualizacija je izvajanje večih navideznih strojev na eni strojni opremi. Omogoča boljšo izrabo strojnih virov, saj strežniki tipično uporabljajo le 10-15% razpoložljivih virov.

Virtualizator je programska oprema, ki omogoča delovanje več navideznih računalnikov (virtualnih strojev) na enem fizičnem računalniku. Vsak virtualni stroj deluje kot ločen sistem z lastnim operacijskim sistemom.

Kakšne vrste virtualizatorjev poznamo?

Virtualizatorje lahko razdelimo na dve vrsti:

  • Kot operacijski sistem (Tip 1) – delujejo neposredno na strojni opremi in sami upravljajo virtualne stroje (npr. VMware ESXi, Hyper-V Server).
  • Kot programska oprema (Tip 2) – namestijo se znotraj obstoječega operacijskega sistema kot običajen program (npr. VirtualBox, VMware Workstation).
Kakšne vrste omrežnih vmesnikov poznamo pri virtualizaciji?

Pri virtualizaciji poznamo več vrst omrežnih vmesnikov:

  • NAT vmesnik: omogoča navideznemu stroju dostop do zunanjega omrežja, ne omogoča pa drugim napravam dostopa do navideznega stroja.
  • Gostiteljski vmesnik: Omogoča povezavo med gostiteljem in navideznim strojem, ne omogoča pa povezave z zunanjim omrežjem.
  • Povezan vmesnik: Omogoča vse povezave - med gostiteljem, navideznim strojem in zunanjim omrežjem.
APLIKACIJSKA PLAST ▼
Kaj je aplikacijska plast?

Aplikacijska plast je:

  • 7. plast ISO/OSI modela
  • 5. plast TCP/IP modela (sejna + predstavitvena + aplikacijska)
  • Vmesnik med uporabniškim programom in omrežjem
Katere kategorije aplikacijskih protokolov poznamo?

Aplikacijske protokole lahko razdelimo na:

  • Podpora omrežju: DHCP, DNS
  • Oddaljen dostop: Telnet, SSH, RDP/UNC
  • Prenos datotek: FTP, TFTP
  • Hiperbesedila: HTTP, HTTPS
  • E-pošta: SMTP, IMAP, POP3
  • Avtentikacija: RADIUS, LDAP
  • Omrežna shramba: SMB, NFS
DNS ▼
Kaj je DNS in čemu služi?

DNS (Domain Name System) je sistem, ki prevaja domenske naslove v IP naslove.

Domene si lažje zapomnimo kot IP naslove, zato DNS omogoča uporabniku prijaznejše naslavljanje v omrežju.

Na svetu imamo 13 korenskih DNS strežnikov, ki poznajo naslove TLD strežnikov (top level domain).

DNS deluje na 53 vratih transportne plasti z UDP protokolom.

Kakšne tipe DNS vnosov poznamo?
  • A → IPv4 naslov
  • AAAA → IPv6 naslov
  • CNAME → Canonical name (drugo ime) - vnos ima IP naslov druge domene
  • MX → Mail exchange - vnos za e-poštni strežnik
  • PTR → Pointer - prevajanje IP naslova v domeno
  • NS → Name server - podrejeni DNS strežnik
Kakšne vrste DNS strežnikov poznamo?

Poznamo dve glavni vrsti DNS strežnikov:

  • Avtoritativni DNS strežniki - skrbijo za 1 domeno
  • Dostopni DNS strežniki - njih uporabljajo uporabniki za hitrejši prevod domene

Dobro znani dostopni DNS strežniki:

  • ARNES: 193.2.1.66, 193.2.1.72
  • GOOGLE: 8.8.8.8
  • CLOUDFLARE: 1.1.1.1
DHCP ▼
Kaj je DHCP?

DHCP (Dynamic Host Configuration Protocol) je protokol za avtomatsko dodeljevanje IP naslovov napravam v omrežju.

Deluje na UDP vratih 67.

Kako poteka pridobitev IP naslova preko DHCP?
  1. DHCP discover - odjemalec pošlje paket vsem napravam v omrežju (broadcast: FF:FF:FF:FF:FF:FF), kjer sprašuje, če obstaja DHCP strežnik
  2. DHCP offer - DHCP strežnik "iskanju" odgovori s ponudbo, ki vsebuje informacije o omrežju (IP naslov, maska, prehod...)
  3. DHCP request - odjemalec strežnik prosi za dodelitev IP naslova
  4. DHCP acknowledge - DHCP strežnik potrdi izdajo naslova in nastavi čas veljavnosti
Kako nastavljamo DHCP strežnik?

DHCP strežnik ima nastavljene bazene (pool) za posamezno omrežje.

Obvezne lastnosti:

  • Omrežni naslov → katere IP naslove uporablja
  • Maska

Neobvezne lastnosti:

  • Privzeti prehod
  • DNS strežnik (lahko tudi več)
  • Časovni pas
  • Časovni strežnik NTP
  • Log strežnik
  • Zagonski strežnik PXE
ODDALJEN DOSTOP ▼
Kaj je Telnet in za kaj se uporablja?

Telnet:

  • TCP protokol na vratih 23
  • Protokol za prenašanje 8-bitnih znakov
  • Prvotno mišljen za oddaljen dostop
  • Danes se ne uporablja, ker ni šifriran

Uporablja se za:

  • Razhroščevanje - direktna tekstovna komunikacija s strežnikom
  • Za radioamaterske DX strežnike
Kaj je SSH in katere prednosti ima?

SSH (Secure Shell):

  • Uporablja se za oddaljen dostop do strežnika
  • TCP protokol na vratih 22
  • Povezava je šifrirana → prenesene informacije so neberljive
  • Šifriranje z asimetričnimi ključi, privzete dolžine 2048 bitov

Ponuja več načinov avtentikacije:

  • Z geslom
  • Z uporabo javnega ključa:
    • Na računalniku ustvarite zasebni in javni ključ
    • Javni ključ naložite na strežnik
    • Ob prijavi strežnik preveri vašo verodostojnost z uporabo ključa

Uporaba:

  • Oddaljen dostop do strežnika
  • GIT
  • Prenos datotek (SCP - Secure Copy)
Kaj sta RDP in VNC?

RDP (Remote Desktop Protocol):

  • Za oddaljen dostop do namizja
  • TCP protokol na vratih 3389
  • Microsoftov lastniški protokol
  • Odjemalec: povezava z oddaljenim namizjem

VNC (Virtual Network Client):

  • Za oddaljen dostop do namizja
  • TCP protokol na vratih 5900+
  • Odprtokoden
  • Več različic: RealVNC, TigerVNC, TightVNC
FTP IN PRENOS DATOTEK ▼
Kaj je FTP in kako deluje?

FTP (File Transport Protocol):

  • TCP protokol na vratih 21 (za pošiljanje ukazov) in 20 (za podatke)
  • 2 načina delovanja: Active in Passive

Uporaba:

Grafični vmesnik:

  • FileZilla
  • WinSCP

Konzolni vmesnik - ukazi:

  • DIR - izpiše datoteke
  • CD - change directory
  • PWD - izpiše kje se nahajamo
  • GET - prenos datoteke na računalnik
  • PUT - naloži datoteko
  • DELETE - izbriše datoteko
  • BYE - prekinitev povezave

Danes se za prenos datotek namesto FTP pogosteje uporabljajo:

  • SFTP
  • FTPS
  • SSH (SCP)
HTTP IN HTTPS ▼
Kaj je HTTP in kako deluje?

HTTP (HyperText Transfer Protocol):

  • TCP protokol na vratih 80
  • Namenjen za prikaz/dostop do spletnih strani
  • Deluje po zgledu REST (Representational State Transfer)
  • Za vsako zahtevo nam strežnik vrne odgovor
  • Odgovor se lahko spreminja glede na stanje strežnika
Kakšna je zgradba HTTP zahtevka?

Primer HTTP zahtevka:

GET /index.html HTTP/1.1
Host: www.example.com
...
  • GET - HTTP glagol
  • /index.html - pot do datoteke
  • HTTP/1.1 - HTTP različica
  • Host: - HTTP glava
  • Konec zahteve označujeta 2 prazni vrstici

HTTP glagoli so dejanja, ki jih izvedemo nad spletnim strežnikom:

  • GET - pridobi
  • POST - spremeni
  • PUT - ustvari
  • DELETE - izbriši

HTTP glave:

  • Host - domena (za izbiro prave strani)
  • Accept-language - predstavitvena plast
  • Accept - oblika zapisa
  • Accept-Encoding - stiskanje
Kakšne HTTP statusne kode poznamo?

Primer HTTP odgovora:

HTTP/1.1 200 OK
  • HTTP/1.1 - HTTP različica
  • 200 OK - statusna koda in sporočilo

Pomembnejše HTTP statusne kode:

  • 200 OK - zahteva uspešna
  • 403 Forbidden - dostop zavrnjen
  • 404 Not Found - vsebina ne obstaja
  • Kode 5xx - napake strežnikov
Kaj je HTTPS?

HTTPS (HTTP Secure):

  • TCP protokol na vratih 443
  • Prenos je šifriran
  • Varen prenos občutljivih podatkov
E-POŠTA ▼
Kateri protokoli se uporabljajo za pošiljanje e-pošte?

SMTP (Simple Mail Transfer Protocol):

  • Namenjen pošiljanju e-pošte
  • Nešifrirano: vrata 25, TCP
  • Šifrirano s TLS: vrata 587
  • Šifrirano s SSL: vrata 465
Kateri protokoli se uporabljajo za prejemanje e-pošte?

POP3 (Post Office Protocol):

  • Nešifrirano: vrata 110, TCP
  • Šifrirano SSL/TLS: vrata 995
  • Prenešena sporočila izbriše s strežnika

IMAP (Internet Message Access Protocol):

  • Nešifrirano: vrata 143, TCP
  • Šifrirano: vrata 993
  • Prenešeno e-pošto pusti na strežniku
OMREŽNA SHRAMBA ▼
Kaj je omrežna shramba in katere naprave poznamo?

Omrežna shramba je imenik, ki je preko omrežja dostopen uporabniku. Uporabniku lahko omejimo pravice nad posameznim deljenim imenikom.

NAS (Network-Attached Storage) je računalnik, ki omogoča deljenje imenika preko protokolov za omrežno shrambo.

Vrste naprav:

  1. Namenske NAS naprave:
    • Vgrajen OS, ki omogoča deljenje datotek
    • Nizka poraba - specializirane
    • Manj svobode - slab procesor, malo pomnilnika, dela samo eno stvar
    • Posodobitve so omejene
    • Primeri: Synology, QNAP, Ugreen
  2. Specializiran OS za omrežno shrambo:
    • OS, ki ga naložimo na računalnik
    • Lahko dela več stvari - Linux
    • Bolj zmogljiv
    • Podpira nove datotečne sisteme - ZFS
    • Višja poraba energije - sorazmerno z zmogljivostjo
    • Primeri: TrueNAS, UnRAID
Kaj je RAID in kakšne vrste poznamo?

RAID (Redundant Array of Independent Disks) je tehnologija za povezovanje več diskov za izboljšano zanesljivost ali zmogljivost.

RAID0:

  • Porazdeli podatke čez vse diske
  • Dobro: povečana hitrost pisanja/branja
  • Slabo: če en disk odpove, izgubimo vse podatke

RAID1:

  • Zrcalna kopija podatkov
  • Dobro: redundanca enega diska
  • Slabo: ½ uporabnega prostora

RAID10:

  • Kombinacija RAID0 in RAID1
  • Dobro: hitrost, do 2 diska redundance*
  • Slabo: ½ uporabnega prostora

RAID5, RAID6:

  • Uporabi 1 oz. 2 diska za pariteto
  • Dobro: 1/2 diska redundance
  • Slabo: 2/3 uporabnega prostora

Princip 3-2-1 za varnostne kopije:

  • 3 različne kopije (backup)
  • 2 na različnih medijih
  • 1 od tega na drugi lokaciji
Kateri protokoli se uporabljajo za omrežno shrambo?

SMB (Server Message Block):

  • Windowsov protokol za komunikacijo med napravami
  • Omogoča dostop do deljenih imenikov
  • Izvaja avtentikacijo in avtorizacijo uporabnikov

NFS (Network File Share):

  • Protokol za Linux sisteme
  • Nima avtentikacije, ampak uporablja pravice v datotečnem sistemu (ACL)
LINUX UKAZI ▼
Kako uporabljamo osnovne Linux ukaze za delo z datotekami?

Najpogostejši ukazi za delo z datotekami:

  • ls - izpis vsebine direktorija
  • ls -lat - sortiranje izpisa
  • ls -latr - obrnjeno sortiranje
  • ls -laS - sortiranje po velikosti
  • cat datoteka.txt - izpiše vsebino datoteke
  • ls ?a??.txt - najde datoteke po vzorcu
  • cat ????.txt - izpiše vsebino datotek, ki ustrezajo vzorcu
Kako uporabljamo Linux ukaze za upravljanje z direktoriji?
  • mkdir -p vaja/vaja9 - ustvari direktorij z nadrejeno mapo
  • pwd - izpiše pot do trenutne mape
  • cd imenik - premik v drug direktorij
Kako ustvariti in zagnati program v Linuxu?
  1. nano neki.py - odpre urejevalnik besedila za ustvarjanje programa
  2. python3 neki.py - zažene Python program
  3. where python3 - najde pot do Python interpreterja
  4. Za avtomatsko izvajanje dodamo v prvo vrstico datoteke #! /path/to/interpreter
  5. chmod +x file.py - nastavi dovoljenja za izvajanje
Kako uporabljamo Linux ukaze za vzdrževanje sistema?
  • sudo su - preklop na administratorskega uporabnika
  • apt upgrade - posodobi nameščene pakete
  • apt install paket - namesti določen paket
  • apt search dhcp - išče pakete, ki vsebujejo "dhcp"
  • apt autoremove - odstrani nepotrebne pakete
  • systemctl restart apache2 - ponovno zažene storitev
  • systemctl stop apache2 - ustavi storitev
  • systemctl start apache2 - zažene storitev
  • systemctl disable apache2 - onemogoči samodejni zagon storitve
  • systemctl enable apache2 - omogoči samodejni zagon storitve
Kako preverjamo DNS in omrežne nastavitve v Linuxu?
  • ping url - preveri dosegljivost gostitelja
  • nslookup url - poizvedba o domeni
  • nslookup google.com 8.8.8.8 - poizvedba preko določenega DNS strežnika
  • ip a - izpis omrežnih vmesnikov in IP naslovov
